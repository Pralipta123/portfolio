## Portfolio

This **`docs`** folder exists so Jekyll does not fail with **“No such file or directory … /docs”** when GitHub Pages is set to publish from **`/docs`**.

To serve the **Angular** build instead:

1. Open the repo on GitHub → **Settings** → **Pages**
2. Under **Build and deployment**, choose **Source: GitHub Actions** (not “Deploy from a branch” using `/docs`)
3. Run or push to `main` so the workflow **Deploy to GitHub Pages** completes

Until then, this page is only a placeholder.
